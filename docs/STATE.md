OR-of-row-OR-col matches the "show cells surviving the filter intersection" mental model (the AND-of-passing → OR-of-failing De Morgan flip). Asymmetric with WD2-wiring-rest-rect's AND-dispatch shape by design. Raw-value lookups (`rowRawByKey.get(row)`, `colRawByKey.get(col)`) reuse the maps WD2-wiring-rest-rect already built so non-string dims (Date / numeric / boolean) match correctly. Cell `<rect>` gains `fillOpacity={isDashboardDimmed ? 0.4 : 1}`; the 0.5-px background stroke + title tooltip stay untouched (structural grid lines, not the filterable mark). 13 source-inspection tests across 6 suites in [`wd2DimRect.test.ts`](../client/src/pages/Dashboard/lib/wd2DimRect.test.ts). WD2-dim-* family now spans 7 of 13 visx renderers (bar + cat-5 + rect).
- **WD2-dim-cat** (2026-05-19) · 5 categorical visx renderers (Arc / Funnel / Box / Waterfall / Combo) dim non-matching marks. Each lifts the WD2-dim-bar triplet (`dashboardFilters / xFilterSel / dashboardDimActive`) on the renderer-specific primary x-field (`labelCh.field` for Arc; `enc.x.field` for Funnel / Box / Waterfall; `xCh.field` for Combo) once per render; per-mark `isDashboardDimmed` checks `isCrossFilterActive` against the renderer-specific raw value identifier (`arc.data.rawKey` / `s.rawLabel` / `s.rawCategory` / `b.rawCategory` / `rawX`); each mark's existing `fillOpacity` baseline composes `baseline * (isDashboardDimmed ? 0.4 : 1)`. Two carve-outs mirror the WD2-wiring-rest-cat dispatch carve-outs: Waterfall running-total bars (`b.isTotal`) AND-gated out so synthetic summary rows stay full opacity; Combo secondary-axis `<LinePath>` stays untouched because a continuous trend dimmed by a categorical filter would render as a gappy interleave. Box whiskers + median stroke stay full opacity (structural geometry needed for distribution comparison even on non-matching categories). 27 source-inspection tests across 6 suites in [`wd2DimCat.test.ts`](../client/src/pages/Dashboard/lib/wd2DimCat.test.ts). WD2-dim-* family now spans 6 of 13 visx renderers (bar + cat-5).
- **WD2-dim-bar** (2026-05-19) · BarRenderer dims non-matching bars under active dashboard cross-filter. Imports `isCrossFilterActive` from [`crossFilter.ts`](../client/src/pages/Dashboard/lib/crossFilter.ts); lifts `dashboardFilters = dashboardTile?.filters` + `xFilterSel = dashboardFilters?.[enc.x.field]` + `dashboardDimActive = !!xFilterSel && xFilterSel.type === "categorical" && xFilterSel.values.length > 0` once per render (right after the existing `useDashboardTileContext()` call); inside `cells.map`, `isDashboardDimmed = dashboardDimActive && !isCrossFilterActive(dashboardFilters!, enc.x.field, c.outerRaw)` runs per bar. `fillOpacity` ternary extends from `(isFiltered ? 1 : grid.inGrid && grid.filter ? 0.4 : 1)` to `(isFiltered ? 1 : grid.inGrid && grid.filter ? 0.4 : isDashboardDimmed ? 0.4 : 1)` — chat/explorer branch stays first because `grid.inGrid` and `dashboardTile` are mutually exclusive contexts.
