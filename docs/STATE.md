- **W61-audit-revert** (2026-05-20) · Server-side one-call revert endpoint that consumes the W61-audit-log ring buffer + W61-audit-history-api read surface. The about-to-be-overwritten model is appended to the audit log via `appendSemanticModelAuditEntry` so "undo this revert" works without losing the intermediate state (the round-trip test pins this directly: revert v2→v1, then revert again with `auditEntryIndex: 0` which now targets the just-captured v2 entry, restoring v2 content with version=4 and a 3-entry audit log). **Why W61-source-bump is NOT run on revert** — the bumper's semantic is "admin edited → bump to `user`"; a revert's semantic is "restore as-was", every per-entry `source` should come verbatim from the snapshot. The model-level `updatedBy` still records the reverting admin (preserves attribution at the model level) but per-entry source comes from the snapshot.
