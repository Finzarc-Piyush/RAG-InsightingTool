The `filters` snapshot is the field that DOESN'T exist on `CrossFilterEvent`: drill-through is a server-side query that needs the entire filter context atomically at click time so the server applies all of them BEFORE pinning (column, value); cross-filter dispatch is purely additive (the dashboard re-renders against the new map automatically). **`DRILL_THROUGH_EVENT = "marico:drill-through"`** — canonical CustomEvent name; `marico:` prefix namespaces apart from anything chart libraries dispatch on window. **`isModifierClick(event)`** — cross-platform modifier check. macOS uses `metaKey` (⌘); Windows / Linux use `ctrlKey`; EITHER fires drill-through (OR-truth, not AND — Safari occasionally synthesises both flags on synthetic events, and the contract should accept that). Accepts a sparse event shape `{ metaKey?: boolean; ctrlKey?: boolean } | undefined | null` so the three event shapes used by the WD2-wiring family (React synthetic / native DOM / ECharts onChartClick) all pass typecheck. `undefined` / `null` event → `false` (defensive: a renderer that loses its event in a test / SSR path falls back to the safe cross-filter intent rather than dispatching a malformed drill-through). **`dispatchDrillThrough(event)`** — fires the CustomEvent on `window`; SSR-safe short-circuit (`if (typeof window === "undefined" || typeof CustomEvent === "undefined") return false;`) mirrors `dispatchCrossFilter`. The cmd / ctrl detection MUST be uniform across all 15 renderers (10 visx + 5 ECharts) so the modifier check lives in a single helper rather than duplicated per-renderer — same "share the helper, not the wiring" precedent as WD2's crossFilter.ts. Renderer onClick handler shape (to be wired in subsequent waves): `if (isModifierClick(event)) { dispatchDrillThrough({...}); return; } dispatchCrossFilter({...});` — a clean two-line branch. 11 pure-function tests across 4 suites in [`drillThrough.test.ts`](../client/src/pages/Dashboard/lib/drillThrough.test.ts) (DRILL_THROUGH_EVENT canonical name; metaKey / ctrlKey / both / neither / null / undefined / unrelated `shiftKey` + `altKey` ignored; SSR-safe short-circuit on dispatchDrillThrough; minimal vs. full DrillThroughEvent shape acceptance). Pairs naturally with the now-complete WD2-wiring + WD2-dim families (which provide the click capture + visual feedback respectively); WD3 layers the modifier-key intent on top.

- **WD2-dim-rect** (2026-05-19) · RectRenderer heatmap dims via OR-of-row-OR-col.
