- **WD2-wiring-rest-rect** (2026-05-19) · RectRenderer heatmap cells wired to `dispatchCrossFilter`. A cell sits at the intersection of two categorical dims (`rowCh`, `colCh`); the `onClick` dispatches TWO `CROSS_FILTER_EVENT` events in row-first order, one per dim. `applyCrossFilter` is pure and event-driven, so back-to-back dispatches each toggle their own column independently — the user sees the row + col filter applied, and re-clicking the same cell toggles both back off. The two parallel `Set` builders for the row / col domains collapse to a single `useMemo` returning `{ rows, rowRawByKey, cols, colRawByKey }` so the type-original raw value for each domain entry is recoverable at dispatch time. 11 tests across 2 suites in [`wd2WiringRestRect.test.ts`](../client/src/pages/Dashboard/lib/wd2WiringRestRect.test.ts) source-inspect both dispatches, the row-first ordering via `indexOf` (NOT `search` — that returns the OUTER match start), and the raw-value preservation.
